browseSetup()

Called every time the browse page mounts. Use it to configure how the model's record listing behaves — filters, default sort order, bulk selection, browse action buttons, query modifications, and optional HTML injected around the table. 

// /App/WRLA/ManageableModels/Blog.php (example)

public final static function browseSetup(): void
{
    // Browse filters
    static::setBrowseFilters(
        static::getBrowseFilterSearch(),
        static::getBrowseFilterSoftDeleted(),
    );

    // Remember filters
    static::setRememberFilters(true);

    // Browse actions
    static::setBrowseActions(static::getDefaultBrowseActions());

    // Enable multi selection
    static::setMultiSelect(true);

    // Order by
    static::setOrderBy('id', 'desc');
}

setBrowseFilters()

Registers the filter controls shown above the browse table. Pass any number of BrowseFilter instances (or null values — they are silently ignored), either as individual arguments or nested arrays; WRLA flattens them automatically. 

static::setBrowseFilters(
    static::getBrowseFilterSearch(),
    static::getBrowseFilterSoftDeleted(),
);

getBrowseFilterSearch()

Returns a pre-built text BrowseFilter that searches across every column shown in getBrowseColumns(). The search is case-insensitive and uses LIKE matching. Relationship columns (dot-notation) are handled automatically via orWhereRelation. 

static::setBrowseFilters(
    static::getBrowseFilterSearch(),
);

Customize the filter by chaining setOption() or setOptions() on the returned instance: 

// Merge extra columns into the search (e.g. computed / aliased columns)
static::getBrowseFilterSearch()
    ->setOption('mergeColumns', ['full_name', 'tags.name']);

getBrowseFilterSoftDeleted()

Returns a pre-built select BrowseFilter that lets users switch between active, soft-deleted, and all records. Only available when the underlying Eloquent model uses SoftDeletes — returns null otherwise (safe to always include). 

static::setBrowseFilters(
    static::getBrowseFilterSearch(),
    static::getBrowseFilterSoftDeleted(), // null-safe — no-op if model has no soft deletes
);
Option value Behaviour
not_trashed Active records only (default)
trashed Soft-deleted records only
all All records including soft-deleted

Custom BrowseFilter

Any manageable field type can be turned into a browse filter by calling makeBrowseFilter() on it and chaining browseFilterApply() with a closure. The closure receives the current Eloquent Builder, the table name, the visible column map, and the current filter value. 

use WebRegulate\LaravelAdministration\Classes\BrowseFilter;
use WebRegulate\LaravelAdministration\Classes\ManageableFields\Select;
use Illuminate\Database\Eloquent\Builder;

static::setBrowseFilters(
    static::getBrowseFilterSearch(),

    // Custom "Status" dropdown filter
    Select::makeBrowseFilter('statusFilter')
        ->setLabel('Status', 'fas fa-tag text-slate-400 !mr-1')
        ->setItems([
            'all'       => 'All',
            'published' => 'Published',
            'draft'     => 'Draft',
        ])
        ->setOption('containerClass', 'w-1/6')
        ->browseFilterApply(function (Builder $query, $table, $columns, $value) {
            if ($value === 'all') return $query;
            return $query->where("$table.status", $value);
        }),
);

setDynamicBrowseFilters()

Switches the browse page to a dynamic filter builder mode. Instead of a fixed set of filter controls, the user can add, configure, and combine filter conditions at runtime. Optionally pass an array of default filter configurations that are pre-loaded when the page first opens. 

// Enable dynamic filters (no defaults)
static::setDynamicBrowseFilters();

// Enable dynamic filters with a pre-applied default condition
static::setDynamicBrowseFilters([
    ['field' => 'status', 'operator' => '=', 'value' => 'published'],
]);
Calling setDynamicBrowseFilters() automatically clears any filters previously set with setBrowseFilters() — the two modes are mutually exclusive.

setRememberFilters()

When enabled, the current filter values are persisted in the session so they survive page refreshes and navigation away from the browse page. Defaults to true in the generated stub. 

static::setRememberFilters(true);  // persist filters in session (default)
static::setRememberFilters(false); // reset filters on every page load

setBrowseActions()

Registers the action buttons rendered in the top-right area of the browse page (above the filter bar). Pass any number of Blade view strings, Livewire-aware HTML strings, or view instances; they are merged and sorted by key. The generated stub includes getDefaultBrowseActions() which provides a Create button and an Export CSV button. 

// Use the built-in defaults (Create + Export CSV)
static::setBrowseActions(static::getDefaultBrowseActions());

// Entirely custom actions
static::setBrowseActions(
    static::getDefaultBrowseActions(),
    view('wrla::components.forms.button', [
        'text'  => 'Import',
        'icon'  => 'fa fa-upload',
        'color' => 'secondary',
        'size'  => 'small',
        'href'  => route('admin.import'),
    ]),
);

getDefaultBrowseActions()

Returns a keyed Collection containing the two standard browse actions. Both buttons are permission-aware — they only render if the corresponding ManageableModelPermissions is granted.

Button Required permission Description
Create CREATE Links to the model's create route
Export CSV BROWSE Exports the current filtered result set as a CSV file, with an optional row-count prompt

setMultiSelect()

Enables (or disables) row checkboxes in the browse table. When multi-select is on, a toolbar appears above the table whenever rows are selected, showing any InstanceAction that defines a multiAction() handler. The built-in InstanceActionDelete and InstanceActionRestore both include multi-action support out of the box. 

static::setMultiSelect(true);  // enable row checkboxes (default in stub)
static::setMultiSelect(false); // hide checkboxes

To add a custom bulk action, define a multiAction() on any InstanceAction returned from getInstanceActions(): 

use WebRegulate\LaravelAdministration\Classes\InstanceAction;

// Inside getInstanceActions():
InstanceAction::make($this, 'Publish', 'fa fa-globe', 'primary')
    ->requireCondition($this::getPermission(ManageableModelPermissions::EDIT))
    ->setAction(fn ($model) => 'Row action triggered')
    ->multiAction(function (array $ids) {
        \App\Models\BlogPost::whereIn('id', $ids)->update(['status' => 'published']);
        return count($ids).' posts published.';
    }, 'Publish the selected posts?') // optional confirmation prompt
    ->setMultiActionText('Publish Selected'); // optional toolbar button label override

setOrderBy()

Sets the default column and direction used to sort records when the browse page first loads. Users can override this interactively by clicking column headers. Both arguments default to 'id' and 'desc' respectively. 

static::setOrderBy('id', 'desc');          // newest first (default)
static::setOrderBy('created_at', 'asc');   // oldest first
static::setOrderBy('title');               // alphabetical (direction defaults to 'desc')
setOrderBy() only controls the initial sort. Once a user clicks a column header to change the order, their selection takes precedence for that session.

setPreQuery()

Registers a callable that receives the base Eloquent Builder and the current array of raw filter values, and must return a modified Builder. WRLA applies this before running any of the user-facing filters, so it is the right place for permanent constraints — tenancy scoping, eager loads, additional joins, or computed columns added via selectRaw. 

use Illuminate\Database\Eloquent\Builder;

// Scope records to the authenticated team
static::setPreQuery(function (Builder $query, array $filters): Builder {
    return $query->where('team_id', auth()->user()->team_id);
});

// Eager-load a relationship and add a computed column
static::setPreQuery(function (Builder $query, array $filters): Builder {
    return $query
        ->with('author')
        ->selectRaw('*, CONCAT(first_name, " ", last_name) AS full_name');
});
The second argument $filters is the raw key → value map of every registered browse filter. You can use it to make the base query react to filter state — for example conditionally adding a JOIN only when a relational filter is active.

appendPreQuery()

Like setPreQuery() but stacks on top of any callable already registered instead of replacing it. Useful in base or trait classes where you want to add constraints without overriding subclass queries. 

// In a shared trait used by multiple manageable models:
static::appendPreQuery(function (Builder $query, array $filters): Builder {
    return $query->where('is_active', true);
});

setAdditionalRender()

Injects arbitrary HTML into one of several named positions on the browse page. The second argument is a zero-parameter callable that returns an HTML string. Multiple calls to the same position are supported — each overwrites the previous callable for that position. 

use WebRegulate\LaravelAdministration\Enums\AdditionalRenderPosition;

static::setAdditionalRender(
    AdditionalRenderPosition::BROWSE_BELOW_FILTERS,
    fn () => '
Content is reviewed daily.
',
);

Available positions

Enum case Where the HTML is injected
BROWSE_TOP Very top of the browse page, before the heading
BROWSE_BELOW_HEADING Directly below the page heading / breadcrumb area
BROWSE_BELOW_ACTIONS Below the browse action buttons, above the filter bar
BROWSE_BELOW_FILTERS Below the filter bar, directly above the results table
The callable is executed at render time, so it has full access to the current request, session, and authenticated user — useful for contextual banners or stat summaries.
