mainSetup()

Called once when the manageable model is registered at boot. Use it to define the model's core identity — which Eloquent model it wraps, how it appears in the UI, its URL, permission overrides, and child navigation items. 

// /App/WRLA/ManageableModels/Blog.php (example)

public final static function mainSetup(): void
{
    // Main
    static::setBaseModelClass(\App\Models\BlogPost::class);
    static::setDisplayName();
    static::setUrlAlias('blog-post');
    static::setIcon('fa fa-file-alt');

    // Permissions
    static::setPermission(ManageableModelPermissions::ENABLED, true);
    static::setPermission(ManageableModelPermissions::BROWSE, true);
    static::setPermission(ManageableModelPermissions::CREATE, true);
    static::setPermission(ManageableModelPermissions::EDIT, true);
    static::setPermission(ManageableModelPermissions::DELETE, true);
    static::setPermission(ManageableModelPermissions::RESTORE, true);

    // Navigation
    static::setNavigationItems(fn () => [
        NavigationItemBrowse::make(static::class),
        NavigationItemCreate::make(static::class),
    ]);
}

setBaseModelClass()

Links the manageable model to its underlying Eloquent model. This is the only strictly required call inside mainSetup(). 

static::setBaseModelClass(\App\Models\BlogPost::class);

setDisplayName()

Sets the human-readable name shown in page titles, breadcrumbs, and buttons. Both arguments are optional — if omitted, WRLA auto-generates them from the class name. 

static::setDisplayName();                          // auto: "Blog Post" / "Blog Posts"
static::setDisplayName('Article');                 // "Article" / "Articles" (auto-plural)
static::setDisplayName('Article', 'All Articles'); // fully explicit
Auto-generation converts StudlyCase to title case and pluralises using Laravel's inflector — e.g. BlogPost → "Blog Post" / "Blog Posts".

setUrlAlias()

Defines the URL slug used for this model's admin routes. For example an alias of blog-posts produces /wr-admin/manageable-models/blog-posts. The artisan command pre-fills this with a kebab-case version of your model name. 

static::setUrlAlias('blog-posts');

setIcon()

Sets the Font Awesome icon class displayed next to this model in the sidebar navigation. The artisan command asks for this during generation — browse the full library at fontawesome.com/v6. 

static::setIcon('fa fa-file-alt');

setPermission()

Overrides a specific permission for this model. The second argument can be a static bool or a callable that receives the current user's WRLA data and returns a bool — useful for role-based access control. 

// Boolean
static::setPermission(ManageableModelPermissions::DELETE, false);

// Callable - See UserData class for available methods
static::setPermission(ManageableModelPermissions::DELETE, fn($userData) => $userData->isMaster());

Available permissions

All permissions default to true unless overridden globally in config/wr-laravel-administration.php under default_manageable_model_permissions.

Enum case Description
ENABLED Enables or disables the entire model in the admin panel
CREATE Allow creating new records
BROWSE Allow browsing and listing records
EDIT Allow editing existing records
DELETE Allow deleting records
RESTORE Allow restoring soft-deleted records
SHOW_IN_NAV Show this model in the sidebar navigation
ENABLED acts as a master switch — setting it to false hides the model entirely regardless of other permission values. SHOW_IN_NAV only controls sidebar visibility; the model's routes remain accessible.

Child Navigation

Sidebar sub-links for a model are defined via setNavigationItems(). Pass a Closure to have the items resolved lazily at navigation build time — this avoids any cross-model ordering concerns (every model's URL alias and display name is already set by then) and skips the work entirely when the sidebar isn't rendered.

setNavigationItems()

Defines the sub-links shown under this model in the sidebar. Pass a Closure that returns an array of NavigationItem instances. 

static::setNavigationItems(fn () => [
    NavigationItemBrowse::make(static::class),
    NavigationItemCreate::make(static::class),
    NavigationItem::make(...), // Custom
]);

Helpful methods

Method Description
NavigationItem::make(...) Creates a custom navigation item from a URL, name, and icon.
->showOnCondition(fn() => ...) Hides the item entirely when the callback returns false.
->enableOnCondition(fn($userData) => ...) Disables the item when the callback returns false; return a string to show it as a tooltip.
->openInNewTab() Opens the link in a new browser tab.
NavigationItem::makeDivider(...) Inserts a visual divider, with an optional title and icon.

NavigationItem

You can replace or extend the default children with hand-crafted NavigationItem instances, pointing to any named route — including routes outside the admin panel. 

use WebRegulate\LaravelAdministration\Classes\NavigationItems\NavigationItem;

static::setNavigationItems(fn () => [
    NavigationItem::make(
        url: route('wrla.manageable-models.browse', ['modelUrlAlias' => static::getUrlAlias()]),
        name: 'All Posts',
        icon: 'fa fa-list',
    ),
    NavigationItem::make(
        url: route('wrla.manageable-models.create', ['modelUrlAlias' => static::getUrlAlias()]),
        name: 'New Post',
        icon: 'fa fa-plus',
    ),
]);

NavigationItem::make()

A static factory that also resolves any ManageableModel class strings in the $children array by calling getNavigationItem() on them automatically. 

NavigationItem::make(
    url: route('wrla.manageable-models.browse', ['modelUrlAlias' => static::getUrlAlias()]),
    name: 'All Posts',
    icon: 'fa fa-list',
);

NavigationItemDivider

Inserts a visual horizontal rule between child navigation items. Optionally accepts a title and/or icon to label the divider. 

use WebRegulate\LaravelAdministration\Classes\NavigationItems\NavigationItem;
use WebRegulate\LaravelAdministration\Classes\NavigationItems\NavigationItemDivider;

static::setNavigationItems(fn () => [
    NavigationItem::make(
        url: route('wrla.manageable-models.browse', ['modelUrlAlias' => static::getUrlAlias()]),
        name: 'All Posts',
        icon: 'fa fa-list',
    ),
    new NavigationItemDivider(),               // plain rule
    new NavigationItemDivider('Management'),   // labelled rule
    new NavigationItemDivider('Tools', 'fa fa-wrench'), // labelled + icon
    NavigationItem::make(
        url: route('wrla.manageable-models.create', ['modelUrlAlias' => static::getUrlAlias()]),
        name: 'New Post',
        icon: 'fa fa-plus',
    ),
]);
You can also create dividers via the convenience static factory NavigationItem::makeDivider('Title', 'fa fa-icon').

NavigationItemHTML

Renders an arbitrary HTML/Blade string in place of a standard navigation entry. Useful for badges, counters, or any custom UI you want embedded in the sidebar. 

use WebRegulate\LaravelAdministration\Classes\NavigationItems\NavigationItemHTML;

static::setNavigationItems(fn () => [
    NavigationItemBrowse::make(static::class),
    NavigationItemCreate::make(static::class),
    new NavigationItemHTML('⚠ Under review'),
]);

showOnCondition()

Fluent method available on any NavigationItem. When the callback returns false the item is completely hidden from the sidebar. 

use WebRegulate\LaravelAdministration\Classes\NavigationItems\NavigationItem;

static::setNavigationItems(fn () => [
    NavigationItemBrowse::make(static::class),
    NavigationItemCreate::make(static::class),
    NavigationItem::make(
        url: route('wrla.manageable-models.browse', ['modelUrlAlias' => 'drafts']),
        name: 'Drafts',
        icon: 'fa fa-pencil',
    )->showOnCondition(fn() => auth()->user()?->can('view-drafts')),
]);

enableOnCondition()

Fluent method available on any NavigationItem. When the callback returns false, or a string reason, the item is visible but rendered as disabled. A string return value is displayed as a tooltip explaining why the item is disabled. 

use WebRegulate\LaravelAdministration\Classes\NavigationItems\NavigationItem;

static::setNavigationItems(fn () => [
    NavigationItemBrowse::make(static::class),
    NavigationItemCreate::make(static::class),
    NavigationItem::make(
        url: route('wrla.manageable-models.create', ['modelUrlAlias' => static::getUrlAlias()]),
        name: 'New Post',
        icon: 'fa fa-plus',
    )->enableOnCondition(
        fn() => PostLimit::hasCapacity() ?: 'Monthly post limit reached'
    ),
]);

openInNewTab()

Fluent method available on any NavigationItem. Forces the link to open in a new browser tab — useful when pointing to external URLs or front-end preview pages. 

use WebRegulate\LaravelAdministration\Classes\NavigationItems\NavigationItem;

static::setNavigationItems(fn () => [
    NavigationItemBrowse::make(static::class),
    NavigationItemCreate::make(static::class),
    NavigationItem::make(
        url: route('blog.index'),
        name: 'View on Site',
        icon: 'fa fa-external-link',
    )->openInNewTab(),
]);
Execution order — WRLA calls every model's mainSetup() during boot, but a navigation Closure is only invoked later, when the sidebar is built. By then every model is fully registered, so you can safely reference another model's URL alias or display name from inside the closure without worrying about registration order.
