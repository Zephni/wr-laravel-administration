Child Navigation

Sidebar sub-links for a model are defined inside mainSetup() via setChildNavigationItems(). Pass a Closure to have the items resolved lazily at navigation build time — this avoids any cross-model ordering concerns (every model's URL alias and display name is already set by then) and skips the work entirely when the sidebar isn't rendered. 

// /App/WRLA/ManageableModels/Blog.php (example)

public final static function mainSetup(): void
{
    static::setBaseModelClass(\App\Models\Blog::class);
    static::setDisplayName();
    static::setUrlAlias('blog');
    static::setIcon('fa fa-newspaper');

    // Navigation (resolved lazily at navigation build time)
    static::setChildNavigationItems(fn () => [
        NavigationItemBrowse::make(static::class),
        NavigationItemCreate::make(static::class),
    ]);
}
Breaking change — the previous globalSetup() method has been removed. Move any child-navigation (or other global) setup into mainSetup(). Wrap navigation in a fn () => ... closure so anything referencing other models still resolves safely regardless of registration order.

setChildNavigationItems()

Defines the sub-links shown underneath this model's entry in the sidebar navigation. Accepts any number of NavigationItem instances (or sub-classes), a flat array of them, or any mix — they are flattened automatically. Alternatively, pass a single Closure returning the items: it is stored as-is and resolved lazily when the navigation is built. 

// Eager — items built immediately
static::setChildNavigationItems(...$childNavigationItems);

// Lazy (recommended) — resolved at navigation build time
static::setChildNavigationItems(fn () => [...$childNavigationItems]);

NavigationItemBrowse / NavigationItemCreate

Lightweight factory classes that build the two standard child links — Browse and Create — already wired to the correct admin routes for the given model class. This is what the generated stub uses. Each make() accepts the manageable model class and optional $name / $icon overrides. 

use WebRegulate\LaravelAdministration\Classes\NavigationItems\NavigationItemBrowse;
use WebRegulate\LaravelAdministration\Classes\NavigationItems\NavigationItemCreate;

static::setChildNavigationItems(fn () => [
    NavigationItemBrowse::make(static::class),
    NavigationItemCreate::make(static::class),
]);

The resulting sidebar entry will show two children:

Label Route Icon
Browse wrla.manageable-models.browse fa fa-list
Create wrla.manageable-models.create fa fa-plus

NavigationItem

You can replace or extend the default children with hand-crafted NavigationItem instances, pointing to any named route — including routes outside the admin panel. 

use WebRegulate\LaravelAdministration\Classes\NavigationItems\NavigationItem;

static::setChildNavigationItems(
    NavigationItem::make(
        url: route('wrla.manageable-models.browse', ['modelUrlAlias' => static::getUrlAlias()]),
        name: 'All Posts',
        icon: 'fa fa-list',
    ),
    NavigationItem::make(
        url: route('wrla.manageable-models.create', ['modelUrlAlias' => static::getUrlAlias()]),
        name: 'New Post',
        icon: 'fa fa-plus',
    ),
);

NavigationItem::make()

A static factory that also resolves any ManageableModel class strings in the $children array by calling getNavigationItem() on them automatically. 

NavigationItem::make(
    route: 'wrla.manageable-models.browse',
    routeData: ['modelUrlAlias' => static::getUrlAlias()],
    name: 'All Posts',
    icon: 'fa fa-list',
);

NavigationItemDivider

Inserts a visual horizontal rule between child navigation items. Optionally accepts a title and/or icon to label the divider. 

use WebRegulate\LaravelAdministration\Classes\NavigationItems\NavigationItem;
use WebRegulate\LaravelAdministration\Classes\NavigationItems\NavigationItemDivider;

static::setChildNavigationItems(
    NavigationItem::make(
        route: 'wrla.manageable-models.browse',
        routeData: ['modelUrlAlias' => static::getUrlAlias()],
        name: 'All Posts',
        icon: 'fa fa-list',
    ),
    new NavigationItemDivider(),               // plain rule
    new NavigationItemDivider('Management'),   // labelled rule
    new NavigationItemDivider('Tools', 'fa fa-wrench'), // labelled + icon
    NavigationItem::make(
        route: 'wrla.manageable-models.create',
        routeData: ['modelUrlAlias' => static::getUrlAlias()],
        name: 'New Post',
        icon: 'fa fa-plus',
    ),
);
You can also create dividers via the convenience static factory NavigationItem::makeDivider('Title', 'fa fa-icon').

NavigationItemHTML

Renders an arbitrary HTML/Blade string in place of a standard navigation entry. Useful for badges, counters, or any custom UI you want embedded in the sidebar. 

use WebRegulate\LaravelAdministration\Classes\NavigationItems\NavigationItemHTML;

static::setChildNavigationItems(
    NavigationItemBrowse::make(static::class),
    NavigationItemCreate::make(static::class),
    new NavigationItemHTML('⚠ Under review'),
);

showOnCondition()

Fluent method available on any NavigationItem. When the callback returns false the item is completely hidden from the sidebar. 

use WebRegulate\LaravelAdministration\Classes\NavigationItems\NavigationItem;

static::setChildNavigationItems(
    NavigationItemBrowse::make(static::class),
    NavigationItemCreate::make(static::class),
    NavigationItem::make(
        route: 'wrla.manageable-models.browse',
        routeData: ['modelUrlAlias' => 'drafts'],
        name: 'Drafts',
        icon: 'fa fa-pencil',
    )->showOnCondition(fn() => auth()->user()?->can('view-drafts')),
);

enableOnCondition()

Fluent method available on any NavigationItem. When the callback returns false, or a string reason, the item is visible but rendered as disabled. A string return value is displayed as a tooltip explaining why the item is disabled. 

use WebRegulate\LaravelAdministration\Classes\NavigationItems\NavigationItem;

static::setChildNavigationItems(
    NavigationItemBrowse::make(static::class),
    NavigationItemCreate::make(static::class),
    NavigationItem::make(
        route: 'wrla.manageable-models.create',
        routeData: ['modelUrlAlias' => static::getUrlAlias()],
        name: 'New Post',
        icon: 'fa fa-plus',
    )->enableOnCondition(
        fn() => PostLimit::hasCapacity() ?: 'Monthly post limit reached'
    ),
);

openInNewTab()

Fluent method available on any NavigationItem. Forces the link to open in a new browser tab — useful when pointing to external URLs or front-end preview pages. 

use WebRegulate\LaravelAdministration\Classes\NavigationItems\NavigationItem;

static::setChildNavigationItems(
    NavigationItemBrowse::make(static::class),
    NavigationItemCreate::make(static::class),
    NavigationItem::make(
        route: 'blog.index',
        routeData: [],
        name: 'View on Site',
        icon: 'fa fa-external-link',
    )->openInNewTab(),
);
Execution order — WRLA calls every model's mainSetup() during boot, but a navigation Closure is only invoked later, when the sidebar is built. By then every model is fully registered, so you can safely reference another model's URL alias or display name from inside the closure without worrying about registration order.
