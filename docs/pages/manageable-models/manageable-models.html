Manageable Models

A Manageable Model wraps an Eloquent model and exposes full CRUD management — browse, create, edit, and delete — inside the admin panel. Each one lives in app/WRLA/ and extends the abstract ManageableModel class.

1 Generate with Artisan 

php artisan wrla:manageable-model {?ModelName}

The model name is optional — if omitted the command will prompt you. Sub-directory namespacing is supported using backslash notation (e.g. Admin\User).

The interactive wizard walks you through three optional extras:

  1. Icon — a Font Awesome v6 class string (e.g. fa fa-file-alt). Defaults to fa fa-question-circle.
  2. Create base Eloquent model? — optionally runs php artisan make:model for you.
  3. Create migration? — optionally runs php artisan make:migration for you.

The generated manageable model is placed at app/WRLA/{ModelName}.php.

2 Registration (automatic)

No registration needed. At boot, WRLA automatically scans app/WRLA/ (recursively) and registers every class that extends ManageableModel. Just drop the file in the directory and you're done.

3 Anatomy of the generated stub

The stub contains three static setup methods called automatically by WRLA on registration, plus three instance methods that define what the browse table and edit/create form look like.

Static setup methods

Method Purpose
mainSetup() Core identity — base model class, display name, URL alias, icon, and permission overrides
globalSetup() Navigation children and other global per-model options
browseSetup() Browse-page configuration — filters, ordering, multi-select, and bulk actions

Instance methods

Method Purpose
getBrowseColumns() Columns shown in the browse table
getInstanceActions() Per-row action buttons — edit, delete, restore, duplicate, etc.
getManageableFields() Fields rendered in the create / edit form
Each method is covered in detail in the sub-sections listed under Manageable Models in the left navigation.
