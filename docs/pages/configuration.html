Configuration

After running php artisan wrla:install, the configuration file is published to config/wr-laravel-administration.php. Options are grouped into four sections matching the comments in that file.

Structural

Core wiring — URLs, models, and middleware.

Key Description
base_url URL prefix for the admin panel. Default: 'wr-admin'
models.user Eloquent User model class used throughout WRLA.
models.wrla_user_data Eloquent UserData model class that stores WRLA-specific user meta.
middleware.prepend Middleware class strings prepended to every WRLA request.
middleware.append Middleware class strings appended to every WRLA request.
middleware.commands Middleware applied to WRLA artisan commands.

Security

Auth Routes

Key Description
wrla_auth_routes_enabled Enable or disable WRLA's built-in login, forgot-password, and reset-password routes.

Captcha

Protects the login and forgot-password forms. Set captcha.current to null to disable.

Key Description
captcha.current Active captcha provider. Options: 'turnstile', null (disabled).
captcha.turnstile.site_key CloudFlare Turnstile site key — set via CLOUDFLARE_TURNSTILE_SITEKEY in .env.
captcha.turnstile.secret_key CloudFlare Turnstile secret key — set via CLOUDFLARE_TURNSTILE_SECRET in .env.

Multi-Factor Authentication (MFA)

Set mfa.current to null to disable MFA.

Key Description
mfa.current Active MFA provider. Options: 'pragmarx/google2fa', null (disabled).
mfa.pragmarx/google2fa.title Issuer name shown in the authenticator app. Supports {app.name} placeholder.
To use pragmarx/google2fa, install the required packages first: 
composer require pragmarx/google2fa-qrcode bacon/bacon-qr-code

Rate Limiting

Each key under rate_limiting maps to a route name and is bound to throttle:{route_name} middleware automatically. Default routes: login.post, forgot-password.post, reset-password.post.

Sub-key Description
rule Rate limit identity. Space-separate identifiers to bucket by multiple factors, e.g. 'input:email ip'.
max_attempts Maximum requests allowed within the decay window.
decay_minutes Time window in minutes before the attempt counter resets.
message Error message shown when the limit is exceeded. Supports :decay_minutes placeholder.

Default Validation Rules

Key Description
default_validation_rules Map of manageable field class strings to their default Laravel validation rule strings. Applied automatically unless a field overrides them.

Default Manageable Model Permissions

Defines default permission gates for all manageable models. Each entry in the array uses a ManageableModelPermissions enum value as its key, and either a bool or a Closure receiving $wrlaUserData and returning a bool as its value. Override per-model inside mainSetup(). 

'default_manageable_model_permissions' => [
    ManageableModelPermissions::ENABLED->value  => fn($wrlaUserData) => $wrlaUserData?->isAdmin() ?? false,
    ManageableModelPermissions::CREATE->value   => true,
    ManageableModelPermissions::BROWSE->value   => true,
    ManageableModelPermissions::EDIT->value     => true,
    ManageableModelPermissions::DELETE->value   => true,
    ManageableModelPermissions::RESTORE->value  => true,
]
Key (ManageableModelPermissions::{KEY}->value) Description
ENABLED Whether the model appears in the admin panel at all.
CREATE Allow creating new records.
BROWSE Allow browsing the record list.
EDIT Allow editing existing records.
DELETE Allow deleting records.
RESTORE Allow restoring soft-deleted records.

Developer Tools

Key Description
developer.enable
  • Closure receiving $wrlaUserData, must return bool
  • null falls back to the legacy enable_developer_tools key
  • e.g. fn($wrlaUserData) => $wrlaUserData?->isMaster()
developer.composer.no_dev Environments in which composer update --no-dev is used instead of a standard update.
developer.update.mode
  • 'live' — runs in the background, streams output as it happens
  • 'blocking' — runs synchronously, shows full output on completion

Documentation

Key Description
documentation.enabled Whether the documentation page route is accessible.
documentation.show_link Whether to show the documentation link in the top bar. Accepts a bool or a Closure receiving $wrlaUserData.

General

Appearance

Key Description
title_template Browser tab title format. Use {page_title} as the dynamic placeholder. Default: '{page_title} - WebRegulate Admin'
logo.light Public asset path to the logo shown in light mode.
logo.dark Public asset path to the logo shown in dark mode.

Logging

Three dedicated daily log channels, each independently configurable:

Channel Purpose
wrla-errorsError-level events from within WRLA.
wrla-infoGeneral informational entries.
wrla-eventsDomain events (model upserts, imports, etc.).

Each channel accepts:

Sub-key Description
enabledEnable or disable this channel.
config.driverLaravel log driver (e.g. 'daily').
config.pathAbsolute path to the log file.
config.levelMinimum log level (e.g. 'error', 'info').
config.daysNumber of days to retain daily log files.

WYSIWYG Editors

Key Description
wysiwyg_editors.current Which editor to use for rich-text fields. Options: 'quill', 'tinymce'.
wysiwyg_editors.quill.initialise Raw JavaScript options passed to the Quill constructor (theme, toolbar, modules, etc.).
wysiwyg_editors.quill.class Additional CSS classes added to the Quill editor container.
wysiwyg_editors.quill.image_uploads.filesystem Laravel filesystem disk used when uploading images from the editor.
wysiwyg_editors.quill.image_uploads.path Storage path (relative to the disk root) where uploaded images are saved.
wysiwyg_editors.tinymce.apikey TinyMCE API key — set via TINYMCE_API_KEY in .env.
wysiwyg_editors.tinymce.plugins Space-separated list of TinyMCE plugins to load.
wysiwyg_editors.tinymce.toolbar TinyMCE toolbar definition string.
wysiwyg_editors.tinymce.image_uploads.* Same filesystem / path keys as Quill above.

Users & User Data

Key Description
user_avatar
  • Closure receiving the User model, returns the avatar URL/path
  • null uses the default placeholder image
  • e.g. fn($user) => $user->profile_image
default_user_data Default JSON structure written to new user data records. Contains permissions, settings, and data keys.
user_groups Named Closures, each returning a Collection of User models for that group. Used for dashboard notifications and group-based features.
build_wrla_user_data_id Closure receiving the User model; returns the identifier used to link a user to their UserData record. Defaults to $user->id.

Misc

Key Description
catch_errors.upsert When true, errors during create/edit upsert operations are caught and displayed in the UI rather than throwing an exception.
email_templates.render_mode How email template bodies are rendered. Options: 'markdown', 'html', 'text'.
dashboard.notifications.user_groups Notification feeds shown on the dashboard. Use '@self' for the current user's own notifications, or any key from user_groups.

CSV Imports

Key Description
csv_imports.chunk_sizeRows processed per chunk. Default: 500.
csv_imports.max_failed_reasonsMaximum row-level failure reasons collected before stopping. Default: 10.
csv_imports.ignore_columnsColumn names skipped during import (e.g. ['id']).
csv_imports.upload_rulesLaravel validation rules applied to the uploaded CSV file.

File Manager

file_manager.enabledEnable or disable the file manager.
file_manager.max_charactersMaximum characters read when previewing text-based files.
file_manager.default_filesystemDefault disk to show. null or empty shows all configured disks.

Log Viewer

Key Description
logs.current Log viewer engine to use. Options: 'wrla', 'opcodesio/log-viewer'.
logs.wrla.max_characters Maximum characters loaded when rendering log files in the built-in viewer.
logs.opcodesio/log-viewer.display_within_wrla When true, embeds the Opcodes Log Viewer inside the WRLA layout instead of redirecting. Configure the package in config/log-viewer.php.

Theme & Styling

Themes

Key Description
default_theme Key from the themes array to use as the active theme.
themes.{key}.name Human-readable theme name shown in the UI when multiple themes are available.
themes.{key}.path Subfolder inside resources/views/themes/ containing this theme's Blade views.
themes.{key}.default_mode Initial colour mode when no user preference is saved. Options: 'light', 'dark'.
themes.{key}.no_image_src Placeholder image path used when no image is set on a record.

Colors

The colors array adds or overrides Tailwind colour palettes injected into WRLA's layout. Each key is a colour name and its value is a shade map (50–950). Use uicolors.app/create to generate a full palette from a single hex value.

Palette key Description
primaryBrand / accent colour used for buttons, links, and highlights throughout the panel.
notesColour scale used for note/info elements.
slateExtended slate palette with additional mid-range shades (550, 725, 750, 850) used in the dark sidebar and header.

Custom CSS

Key Description
common_css Raw CSS injected into the <head> of every WRLA page. Malformed CSS can break the layout.
wysiwyg_css Raw CSS injected into the WYSIWYG editor iframe. Useful for matching editor body styles to your front-end typography.
